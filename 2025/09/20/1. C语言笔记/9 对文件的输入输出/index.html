C笔记之文件的输入输出

对文件的输入输出

文件

什么是文件：存储在外部介质上数据的集合

  1. 程序文件：源程序文件(.c)、目标文件(.obj)、可执行文件(.exe)
  2. 数据文件：文件的内容是供程序运行时读写的数据

输入输出是数据传送的过程，又称为流，也叫“数据流”，流表示了信息从 源 到 目的 端的流动

文件分类：

  1. 数据文件分为：ASCII文件（文本文件）、二进制文件（映像文件）

文件缓冲区

ANSI C标准采用**“缓冲文件系统”**处理数据文件

缓冲文件系统：系统自动地在内存区为程序中每一个正在使用的文件开辟一个文件缓冲区

  1. 从内存向磁盘输出数据必须先送到内存中的缓冲区，装满缓冲区后才一起送到磁盘去
  2. 从磁盘向计算机读入数据，则一次从磁盘文件将一批数据输入到内存缓冲区（充满缓冲区），然后再从缓冲区逐个的将数据送到程序数据区（给程序变量）为了节省存取时间，提高效率
  3. 每个文件在内存中只有一个缓冲区，在向文件输出数据时，它就作为输出缓冲区，在从文件输入数据时，他就作为输入缓冲区

文件类型指针

文件类型指针 简称 文件指针

每个被使用的文件都在内存中开辟一个相应的文件信息区，用来存放文件的有关信息（如文件名、文件状态、文件当前位置等）

这些信息是保存在一个结构体变量中的，该结构体类型是由系统声明的，叫FILE

C编译环境提供的stdio.h头文件中有以下文件类型声明：（知道即可）

FILE结构体类型的信息包含在头文件**<stdio.h>**中，程序中可以直接使用FILE类型名定义变量

一般使用指向FILE类型的变量的指针变量，来引用这些FILE类型变量

FILE * fp;

fp指向某个文件的文件信息区（是一个结构体变量），通过该文件信息区中的信息就能够访问该文件。

通常将这种只想文件信息区的指针变量简称为指向文件的指针变量（指向内存中的文件信息区的开头，不是外部介质上的数据文件开头）

常用输入输出函数归纳

EOF（End Of File）

名称是文件结束标志，定义为 -1（不是ASCII码值为-1），可以通过 Ctrl+Z 直接键入

EOF常常用于判断是否到达了文件末尾或输入流末尾

EOF并不是一个实际的字符或整数值，而是一个特殊的标志，用于指示文件结束或输入流结束的状态

scanf函数

scanf函数是有返回值的，一般来说，它的返回值是成功读取的元素个数。但当遭遇读取失败时，它的返回值便是 -1 （也就是它自己返回一个EOF）

而若是一个元素都还没成功读入的时候就遇到了读取失败或EOF，那它直接就会返回-1，不管后面再输入了什么

getchar函数

从标准输入（键盘）读取一个字符，并返回其ASCII值（作为整数），getchar读取输入缓冲区中的下一个字符，并在读取成功后将其从缓冲区中删除

**返回值：**int型，如果发生错误或达到输入的末尾，getchar返回特殊值EOF

putchar函数

将一个字符输出到标准输出（控制台），

参数：以int型传递，但只使用最低有效的8位，因此实际上被视为char类型处理

**返回值：**将写入的字符作为 unsigned char类型返回，如果发生错误，则返回EOF

注意：

scanf和getchar在读取数据时，是从缓冲区内读取数据，而缓冲区位于键盘和scanf，getchar之间

gets函数

函数原型：char* gets(char* str);

从输入缓冲区中读取一个字符串存储到字符指针变量 str 所指向的内存空间

注意点：

  1. 可以输入带空格的字符串
  2. 使用gets时，系统会将最后的回车换行符从缓冲区中取出来，然后丢弃，不需要额外清空缓冲区了

puts函数

函数原型：int puts(const char* str);

正常打印返回0，打印异常返回EOF

打开、关闭文件

打开：为文件建立相应的信息区（用来存放有关文件的信息）和文件缓冲区（用来暂时存放输入输出的数据）

用fopen函数打开数据文件

fopen(文件名, 使用文件方式);

返回一个FILE指针（文件指针），如果打开失败返回NULL

文件使用方式 含义 如果指定文件不存在
“r”（只读） 为了输入数据，打开一个已经存在的文本文件（从文件获取） 出错
“w”（只写） 为了输出数据，打开一个文本文件（向文件输出） 建立一个新的文件
“a”（追加） 向文本文件尾添加数据 出错
“rb”（只读） 为了输入数据，打开一个二进制文件 出错
“wb”（只写） 为了输出文件，打开一个二进制文件 建立一个新的文件
“ab”（追加） 向一个二进制文件尾添加数据 出错
“r+”（读写） 为了读和写，打开一个文本文件 出错
“w+”（读写） 为了读和写，创建一个新的文本文件 建立一个新的文件
“a+”（读写） 打开一个文件，在文件末尾读进行读写 建立一个新的文件
“rb+”（读写） 为了读和写打开一个二进制文件 出错
“wb+”（读写） 为了读和写，新建一个二进制文本文件 建立一个新的文件
“ab+”（读写） 打开一个二进制文件，在文件末尾进行读和写 建立一个新的文件

如果使用任何一种 “w” 模式打开一个现有文件，该文件的内容会被删除，以便程序在一个空白文件中开始操作

如果不希望删除原有数据，则应用”a”方式打开，打开文件时，文件读写位置标记到文件末尾

用”r+”、”w+”、”a+”方式打开的文件既可输入也可输出

  1. “r+”：该文件应该已经存在，
  2. “w+”：建立一个新文件，先往里写，然后可以读里面的数据
  3. “a+”：原来的文件不被删去，文件读写位置标记到文件末尾，可添加可读

常用的文件读取方式：（加上了一个判断）

**文件使用方式中，带b和不带b的区别：**b表示二进制方式，带不带只有对换行的处理的区别

  • 如果使用文本文件并用”w”方式打开：在向文件输出时，遇到换行符'\n'时，系统就将其转换为'\r'和'\n'两个字符
  • 如果使用文本文件并用”r”方式打开：遇到'\r'和'\n'两个连续的字符是，就转换为'\n'一个字
  • 如果用”wb”方式，并不代表着把内存中的文本文件的数据自动转换成二进制形式存储，输出的数据形式是由程序中采用什么读写语句决定的
  • 根据需要，如果需要对回车符进行转换的，就用”w”，不需要就用”wb”，带b是通知编译系统，不必进行回车符的转换，如果是文本文件（显然需要转换），就用”w”方式
  • 理论上说，文本文件可以用”wb”方式打开，但没必要

程序中可以使用3个标准的 流文件：标准输入流(stdin)，标准输出流(stdout)，标准出错输出流(stderr)

程序开始运行时，自动打开这3个标准流文件（不用手动打开），其中stdin,stdout,stderr是对应的文件指针变量，可通过这三个指针变量对其进行操作

用fclose函数关闭文件

标准形式：fclose(文件指针);

  • 不关闭文件就结束程序运行，数据会丢失
  • fclose先把缓冲区中的数据输出到磁盘文件，然后才撤销文件信息区

顺序读写数据文件

向文件读写字符

函数名 调用形式 功 能 返 回 值
fgetc fgetc(fp) 从fp指向的文件读入一个字符 读成功，带回所读字符
fputc fputc(ch,fp) 把字符ch写到文件指针变量fp所指向的文件中 输出成功，返回值就是输出的字符

失败则返回文件标志EOF (即$-1$)

例：从键盘输入一些字符，并逐个送到磁盘中，直到输入“#”为止

注意：

  1. 用fopen打开一个“只写”文件，如果成功，函数返回值是该文件所建立的信息区的起始地址
  2. exit函数需要<stdlib.h>头文件
  3. 第七行输入文件名后回车，回车仍在缓冲区中，所以用一个getchar将它读出来消化掉，避免回车符被当成有效数据

例2：将一个磁盘文件中的信息复制到另一个磁盘文件中，file1.dat->file2.dat

向文件读写字符串

函数名 调用形式 功 能 返 回 值
fgets fgets(str,n,fp) 从fp指向的文件读入一个长度为(n-1)的字符串
存放到字符数组str中		 读成功，返回地址str，失败返回NULL
fputs fputs(str,fp) 把str所指向的字符串写到文件指针变量fp所指向的文件中 输出成功，返回0,；否则返回非0值

fgets函数：

函数原型：char* fgets(char* str, int n, FILE* p);

注意点：

  • 该函数只会得到n-1个字符，因为要在最后加一个'\0'
  • 遇到'\n'、EOF，读入即结束，'\n'也作为一个字符输入
  • 函数成功，返回字符数组的首元素地址

fputs函数：

函数原型：int fputs(char* str, FILE* fp);

注意点：

  • 函数中第一个参数可以是字符串常量、字符数组、字符型指针
  • 字符串末尾的’\0‘不输出
  • 成功返回0，失败返回EOF（-1）

fgets和fputs和gets、puts很像，但gets/puts是以终端为读写对象，而fgets/fputs函数以指定的文件作为读写对象

用格式化方式读写文本文件

与printf、scanf相同，格式化输出fprintf、fscanf只是多了个f，前两个作用对象是终端，后两个作用对象是文件

int fprintf(文件指针,格式字符串,输出表列);

int fscanf(文件指针,格式字符串,输出表列);

fprintf()的返回值是输出的字符数,发生错误时返回一个负值

用二进制方式向文件读写一组数据

一次输入一组数据，可以用fread函数从文件中读一个数据块，用fwrite函数向文件写一个数据块，读写时是以二进制形式进行的

fread(buffer,size,count,fp);

fwrite(buffer,size,count,fp);

  • buffer：指向存储读取数据的缓冲区的指针
  • size：要读写的字节数
  • count：要读写多少个数据项（每个数据项长度为size）
  • fp：FILE类型指针

fread()和fwrite()主要用于处理二进制文件，因为它们直接读写原始数据，而不进行任何格式转换

如果执行成功，则函数返回形参count的值，即输入或输出的数据项的个数

总结

数据的存储方式

  1. 文本方式：数据以字符（ASCII码）存储到文件中（所以int型数据也只占2字节）
  2. 二进制方式：数据按内存的存储状态原封不动地复制到文件，（int型数据送到文件时占4个字节）

文件的分类

  1. 文本文件：文件中全部为ASCII字符
  2. 二进制文件：按二进制方式把在内存中的数据复制到文件

文件的打开方式

  1. 文本方式：不带b的方式，读写文件时换行符进行转换
  2. 二进制方式：带b的方式，读写文件时不对换行符进行转换

文件读写函数

  1. 文本读写函数：用来向文本文件读写字符数据的函数（fgetc,fgets,fputc,fputs,fscanf,fprintf）
  2. 二进制读写函数：用来向二进制文件读写二进制数据的函数（getw,putw,fread,fwrite）

C语言不禁止文本方式与二进制方式之间存在交叉，虽然合法，但往往会导致出错

随机读写数据文件

文件位置标记及其定位

位置标记

一般情况下，在对字符文件进行顺序读写时，文件位置标记指向文件开头，读一个后就指向后一个位置

可以根据需要，人为的移动文件位置标记，这就是随机读写

对流式文件可以进行顺序读写，也可以进行随机读写，区别就是文件位置标记的移动的规律

位置标记的定位

rewind函数

使文件位置标记重新返回文件的开头，此函数没有返回值

fseek函数

fseek(文件类型指针, 位移量, 起始点)

起始点用0、 1、 2代替

0代表“文件开始位置”，1代表“当前位置”，2代表“文件末尾位置”

位移量：以“起始点”为基点，向前移动的字节数，位移量应是long型数据（数字末尾加L）

fseek函数一般用于二进制文件

ftell函数

得到流式文件中文件位置标记的当前位置（相对于文件开头的位移量）

如果调用出错，返回值为-1L

随机读写

文件读写的出错检测

ferror函数

在调用各种输入输出函数时，如果出错，可以用ferror函数检查

ferror(fp);

如果返回 0(假)，表示未出错；返回非零值，则表示出错，初始值为 0

对同一个文件每次调用输入输出函数，都会产生一个新的ferror函数值

clearerr函数

使文件出错标志和文件结束标志为 0

只要文件出现读写出错标志，它就一直保留，直到对同一文件使用clearerr函数或rewind函数，或任何其他一个输入输出函数
