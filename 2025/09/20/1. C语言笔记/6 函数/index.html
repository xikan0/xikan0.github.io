C笔记之函数

函数

概述

函数就是功能，每一个函数实现一个功能，函数是构建C语言程序的基本单元

一个C程序由一个或多个程序模块组成，每一个程序模块作为一个源程序文件，即一个C程序由若干个源程序文件组成的

  1. 一个源程序文件由一个或多个函数、及其他有关内容组成
  2. 一个源程序文件是一个编译单位（程序编译时，不是以函数为单位进行编译的）
  3. C程序的执行从main函数开始，调用其他函数后返回到main函数，在main函数中结束整个程序的运行
  4. 函数不能嵌套定义，所有函数都是平行的，即在定义函数时是分别进行的，是相互独立的
  5. 函数间可以相互调用，但是不能调用main函数，main函数是被系统调用的（主调函数、被调函数）

分类

函数可分两种：

  1. 库函数：系统提供的，可直接使用
  2. 用户自定义函数

从函数形式上，分两类：

  1. 无参函数：在调用时不需要参数
  2. 有参函数：在调用时，需要参数进行传递数据

从作用范围上看，分为：

  1. 外部函数（extern）
  2. 内部函数（static）
  3. 一般情况，函数默认为 extern 型

定义函数

函数体+函数头

函数体包括：声明部分、语句部分

C语言中，函数必须先定义，后调用

空函数是之后可能要实现的功能，占个空位，或准备补充

内部函数

只能被本文件中的其他函数调用，用static在最前面修饰（也叫静态函数）

外部函数

可以被其他文件的函数调用，用 extern修饰，啥都没加默认是外部函数

调用函数（重点）

定义的函数，其括号内的变量名称为形式参数，又叫虚拟参数

调用时，函数名括号内的变量名称为实际参数，通常是常量、变量或表达式

在调用函数时，系统会把实参的值传递给被调用函数的形参

形参从实参得到一个值

这个传递过程称为**“虚实结合”**

函数的调用过程

  1. 未出现函数调用时，其函数的形参，不占存储单元
  2. 调用时，将实参传递给形参
  3. 函数利用形参进行有关计算
  4. 通过return语句将函数值带回主调函数（不一定是main函数），如果函数不需要返回值，则类型应定义为 void 类型
  5. 调用结束，形参单元被释放

实参、形参的传值是“值传递”，单向传递，实参无法得到形参的值，实参的值也不会因形参被改变

重点：参数传递是单向传递（拷贝）（只有这一种，没有例外）

函数的返回值

函数通过函数中的 return 语句获得返回值

如果不需要从被调用函数中获得返回值，则不需要加 return 语句，return 语句是不必要的

函数类型决定返回值的类型

对于不带返回值的函数，应定义为 void 类型（空类型）

对被调用函数的声明和函数原型

调用库函数时，必须在文件开头用#include 调用它

当被调函数在主调函数后面时，应进行声明

声明：把函数名、函数参数的个数和参数类型等信息通知编译系统，以便遇到函数调用时，编译系统能正确识别函数并检查调用是否合法

例：

函数声明就只比函数定义多了个分号，函数的首行称为函数原型

使用函数原型作声明，是C语言的一个重要特点

声明时可以省掉形参名

声明时，形参名可以不同

可以在文件开头将所有函数声明，这样就不必在主调函数中重复声明了

函数的嵌套调用和递归

汉诺塔问题

数组作为函数参数

当数组元素作为参数时，进行的是值传递

当数组名作为参数时，传递的是数组的首元素地址，因此形参可以不指定大小

多维数组名作函数参数

可以用多维数组名作为实参和形参，在被调用函数中对形参组定义时可以指定每一维的大小，可以省略第一维的大小说明，但第二维不行

局部变量和全局变量

建立存储空间的声明称为定义（定义性声明），不建立存储空间的声明称为声明

变量按照作用域的不同分为局部变量和全局变量

局部变量

只在本函数范围内有效，此函数以外不能使用，在复合语句内定义的变量，只在本复合语句中有效

  1. 在函数开头定义
  2. 在函数内的复合语句内定义
  3. 在函数的外部定义

形参也是局部变量

全局变量

在函数外部定义的变量称为外部变量，也叫全局变量，有效范围是从定义的位置到本源文件结束

函数内定义的就是局部变量，函数外定义的就是全局变量

全局变量不一定所有函数都能使用，注意作用域

作用：增加了函数间数据联系的渠道

全局变量的缺点：

  1. 在程序的全部执行过程中都占用存储单元
  2. 使函数的通用性降低了
  3. 全局变量过多，会降低程序的清晰性

变量的存储方式和生存期

根据变量值存在的时间，可将变量的存储方式分为：

  1. 静态存储方式：程序运行期间，由系统分配固定存储空间
  2. 动态存储方式：程序运行期间，根据需要进行动态分配存储空间

内存中，供用户使用的存储空间有：

  1. **程序区：**存储可执行文件
  2. **静态存储区：**存储静态变量（全局变量，静态局部变量）
  3. **动态存储区：**分为堆区和栈区
    • **堆区：**供用户进行动态内存分配
    • **栈区：**用来存储动态数据，局部自动变量、函数形参、函数调用时所需的临时变量

静态存储区

用于存放程序中的全局变量、静态变量，在程序运行之前为变量分配内存，程序终止前才被释放其内存

动态存储区

划分为 堆 和 栈，“栈” 是局部变量的存储内存区，栈内存由系统自动分配和释放，“堆” 内存需要程序员调用内存管理函数进行分配和释放

里面存放：

  1. 函数形参
  2. 函数中定义的没有用 static 声明的变量，即自动变量
  3. 函数调用时的现场保护和返回地址等

在进行多次调用时，分配给这些局部变量的存储空间地址可能是不相同的

局部变量的存储类别

在C语言中，每一个变量和函数都有两个属性：数据类型和数据的存储类别，存储类别指的是数据在内存中的存储方式（静态存储或动态存储）

C的存储类别有四种：自动的（auto）、静态的（static）、寄存器的（register）、外部的（extern）

1.自动变量（auto变量）

函数中的局部变量，如果没有专门声明为 static ，则都是动态地分配存储空间的，存储在动态存储区中

在调用函数时，函数的形参、内部的局部变量会由系统分配存储空间（在栈内分配），函数调用结束时自动释放

auto 关键字可省略，不写则隐含指定为“自动存储类别”

2.静态局部变量（static局部变量）

静态局部变量属于静态存储类别，在静态存储区内分配存储单元，

注意：

  1. static 局部变量整个程序运行期间都不释放
  2. 对静态局部变量是在编译时赋初值的，即只赋初值一次，以后每次调用函数都不再重新赋初值，而是保留上次函数调用结束时的值
  3. 如果定义局部变量时不赋初值
    • 静态局部变量：自动赋值0（数值型变量）或 '\0' （字符变量）
    • 自动变量：是一个不确定的值
  4. 其他函数不能引用静态局部变量，它只是局部变量

3.寄存器变量（register变量）（了解）

一般情况下，变量的值是存在内存中的，但一些变量使用频繁，允许将局部变量的值存放在CPU中的寄存器中，用关键字 register 进行声明，现在的编译器能自动优化，无需特别声明register

全局变量的存储类别

全局变量都是存在静态存储区中的

外部变量的作用域：从变量的定义处开始，到本程序文件的末尾

1.extern 外部变量声明（一个文件内扩展外部变量的作用域）

由于外部变量定义之前的程序不能使用该变量，可以用外部变量声明将下面的外部变量作用域扩展到这（类似于函数声明）

声明时可以省略类型，只有关键字和标识符

  1. 将外部变量的作用域扩展到其他文件

如果一个文件内已定义了一个外部变量Num，则另一个文件内可以声明 extern Num; 就可以引用

3.将外部变量的作用域限制在本文件中

将全局变量用 static 关键字修饰，称为静态外部变量

注意：

  1. 声明存储类型，局部变量和全局变量的含义不同
  2. 局部变量声明存储类型：指定变量的存储区域（静态存储区或动态存储区）
  3. 全局变量声明存储类型：变量作用域的拓展问题（全是静态存储区）

static声明的注意点：

  1. 凡是有 static 声明的，作用域都是有限的
  2. static 对局部变量和全局变量的作用不同

变量的生存期

变量值存在时间的长短

作用域是空间的角度，生存期是时间的角度

全局变量和静态变量的生存期都是程序级，即整个程序运行期间一直存在

概念：如果一个变量值在某一时刻时存在的，则认为这一时刻属于该变量的生存期，或称该变量此时此刻“存在”
