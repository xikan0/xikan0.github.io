指针

指针的概念

指针是一种特殊的数据类型，用于表示内存地址，（存放另一个变量的地址的变量）

内存的基本单位是字节，每个字节由8位二进制位（bit）组成，内存是连续的编址空间，每一个字节有唯一的地址标识

  1. 指针是无符号整数
  2. 指针的运算不同于整型
  3. 变量定义后，该变量的地址是指针常量
  4. 完整的指针类型包括基类型

指针与整数的区别：

从形式上看，地址（指针）是整数，但指针是无符号整数，其表示范围由系统的寻址空间所确定，而不一定与整形相同，操作上，指针运算不同于整形，例如指针能解引用（*）运算

常使用 %p 格式符来输出十六进制形式的地址

**指针的基类型：**指定存储对象的类型

例：int *p; 则称int为指针p的基类型，或指针p所指向的变量的类型

指针的声明形式：

&的实质（扩展，了解）

使用&时，返回的是一个“指向T的指针”类型的常量，即指针常量

当要给指针赋值，赋一个绝对地址时，通常先将地址写成整形常量的形式，然后强转成相应的类型，如：int*、double*（因为C语言中没有方法去表示指针常量）

但一般来说：除了0之外，整数不能赋给指针变量

* 解引用操作符

*在定义时用来说明一个变量是指针，当使用指针时，*p表示的是p所指向的对象（即i）（对于一个已定义的指针使用 * 操作符，将访问这个指针所指向的对象）

指针变量的初始化

一个指针变量未被初始化时，指向的位置不确定，被称为野指针

**空指针常量：**一个具有0值的整形常量表达式，或者此表达式被强转为void*类型

例：将 0、 0L、 ‘\0’、 2-2、 0*5、 (void*)0 赋给任意类型的一个指针，该指针就成为了空指针

由系统保证空指针不指向任何对象或函数

C 还定义了一个宏 NULL，用来表示空指针常量

对指针进行初始化的几种方式：

在C语言中，将一个字符串常量赋给一个字符指针时，字符指针中存储的是字符串常量的首地址。字符串常量在内存中是连续存储的，并且以字符’\0’结尾，字符指针指向字符串的起始位置

对指针进行初始化或赋值的实质是将一个地址或同类型（或兼容的类型）的指针赋给它，而不管这个地址是怎么取得的

对于一个不确定要指向何种类型的指针，定义之后最好初始化为NULL

void* 型指针

表示定义了一个指针，但不指定它指向何种类型的数据，有基类型，基类型是 void

void* 型指针作为一种通用的指针，可以和其他任何类型的指针（函数指针除外）相互转化而不需要强转，但无法进行解引用或下标操作

不能进行解引用的：野指针、空指针、void*型指针

多级指针（指向指针的指针）

常用的是二级指针

i 的地址为 108，pi 的内容就是 i 的地址，而 pi 的地址为 104，ppi 的内容即是 pi 的地址。对 ppi 解引用照常会得到 ppi 所指的对象，所获得的对象是指向 int 型变量的指针 pi 。想要真正地访问到 i ，必须对 ppi 进行两次解引用

常量指针与指针常量（了解）

常量指针：指针本身的值不可被改变，int *const a;

指针常量：一个指针类型的常量，(int*)0x123456ff

指向const的指针：一个指针它所指向的是一个const对象，const int *p，表明p是一个指向const int 型变量的指针，p自身的值可以被改变，但不能通过对p解引用来改变所指的对象的值

注意：const int 表示“ 指向一个具有const限定符的int类型的指针 ”，即const所修饰的是指针所指向的类型，而非指针

指针与函数

函数的局限性：

  1. 函数调用只能是单向“传值”的方式，被调用的函数无法访问主调函数中定义的对象
  2. 函数只能返回唯一的函数值
  3. 只能向函数传值，不能“传操作”，即参数不能是函数

指针作函数参数

  1. 变量名作实参时，传递的是变量的值，称为按值调用
  2. 变量的地址作实参，传递的是变量的地址，称为按地址调用

易错点：

1. C语言只有唯一的传参数方式，即单向传值

传地址也是单向传值，更改形参的值（地址），主调函数里的实参的值（地址）没关系，不过他俩解引用都指向同一个地址，所以可以用来更改同一个变量

2. 函数只能有一个返回值，但可通过指针来获取函数的多个计算结果

函数指针与指针函数

指针函数：

本质是一个函数，该函数的返回值是一个指针

声明格式为：类型标识符 *函数名(参数表)

解析类型声明需要遵循“从右往左”的顺序，先解释代表函数的（参数表），即它是一个函数，再看函数的原型为“类型 *”

可以将返回值定义为 void* 型，接收时需要强转成自己需要接收的类型

函数指针：（指针常量）

本质是一个指针变量，该指针指向这个函数（就是指向函数的指针）

声明格式：int (*fun)(int x,int y);

函数指针需要把一个函数的地址赋给它：（这样就可以解引用这个指针来调用这个函数）

&地址运算符不是必须的，因为一个函数标识符就表示了它的地址，如果是函数调用，还必须包含一个圆括号括起来的参数表

指针的算数运算

指针来访问数组中的元素

注意：

  1. 在C语言中，访问数组元素本质就是间接访问
  2. 访问数组元素p[i]，p不一定是数组名，p是指向该数组内任意一个元素的指针即可，但那样下标的范围就不是0 ~ n-1 了，而是p所指向的那个元素为原点0，左边下标是负数，右边下标是正数

指针+整数 或指针-整数

指针 + 整数 = 另一个指针

如：上述的 cp + 1，则产生的指针指向数组中的下一个字符，a[i]==*(a+i)

当指针相加减时，所做的就是指针加上或减去步长乘以那个整数的积，步长就是指针所指向的类型的大小（即指针移动一个位置时要跳过几个字节）

注意：

  1. 原来的指针和计算后的新指针都需要指向同一个数组的元素、或指向数组数组范围的下一位置时才是合法的
  2. 自增、自减运算符和*取地址运算符、&的优先级相同，且都是按照自右向左方式结合

指针 - 指针

注意：C标准里，没有定义两个指针相加的运算，所以会报错

**指针的减法运算是有条件的：**当两个指针指向同一数组、或有一个指针指向该数组末端的下一位置时，两个指针可做减法运算

  1. 两指针相减，得到的是两指针所指向元素之间的距离，也就是他们相隔几个数组元素，与步长没关系
  2. 指针可以加减0，指针保持不变，空指针也可以加减0，
  3. 两个空指针相减，结果也是0

指针的关系运算

两指针可进行关系判断：==、!=、<=、>=

判断大小是根据在数组中的相对位置来判断的，如：q[2] <= q[4]

指针与数组

数组名

数组名代表该数组的第一个元素地址（元素的起始地址），数组名是一个常量指针（常指针），自身的值不能改变

在对数组名实施*&*操作时，返回的是一个指向数组的指针，而非具有某个指针常量值的指针

数组和指针的差别：

  • 声明数组时，同时分配了用于容纳数组元素的空间
  • 声明指针时，只分配了用于容纳指针本身的空间

通过数组名引用元素

解释：

  1. ia是数组名，也就是一个指向首个元素的常量指针：ai[0]
  2. 指针可以加整数，返回一个新的指针，指向该数组内的元素的地址
  3. *(ai+99)也就是从 ai[0] 开始向后99位，也就是 ai[99]
  4. 数组名是常指针，不能对它的值进行修改。ia + 99是可以的，但ia++是不行的，它的意思是ia = ia +1;修改了ia的值

函数形参中的数组

一、声明函数形参，数组会被改写成指向数组第一个元素的指针

当数组名作为函数形参时，已经被转换成了一般的指针（不是常指针了），因此，这个形参数组名的值可以被改变了。此时，不能通过sizeof运算符获取数组的长度，不能判断数组的长度时，可能会产生数组越界访问。因此传递数组名时，需要一起传递数组的长度。

C语言中，所有非数组形式的实参（包括指针）均以值传递的形式调用（形参值的变换不影响实参）

例如：主函数传了一个指针，给一个函数的形参，形参拿到的指针是原指针的副本，无法影响实参，但都指向一个地址，可以影响地址里的值

二、用 sizeof(形参数组名)，来获得数组的大小时，只能得到一个指针的大小（因为形参数组名只是一个指针了）

  • 32位系统中，指针大小通常为4字节
  • 64位系统中，指针大小通常为8字节

指向数组的指针(二维数组)

指向数组的指针主要用来对二维数组进行操作

用指针定义二维数组

数组指针：标识符类型 (*标识符)[整形常量]

二维数组名的类型为 数组指针 类型

其中，*p 指向的是数组的第一个元素，而后面的 [100] 下标，则表示这一个元素里又有个长度为100的整形数组，(*p)[100] 就代表每个元素都是一个 int[100] 型的数组，通过对 p 的算数运算可以使它指向 m 数组中的任意一个元素，而不需要指导 m 是一个多长的数组， 但一定要知道 m 中每个数组元素的长度

所以这个二维数组的步长：p + n，也就是 n*100*sizeof(int) 个字节，相当于跳过矩阵中的 n 行，每行都有 100 个 int 型元素

数组越界的问题：

对指针的算术运算可以使它指向数组中的任何一个元素包括超出数组范围的第一个元素，这个超出范围的第一个元素实际上是不存在的，这个“元素”的地址在数组所占的内存之后，它是数组的第一个出界点

这个地址可以赋给指向数组元素的指针，但ANSI C仅允许它进行赋值或比较运算，不能对保存这个地址的指针进行解引用或下标运算。

在表达式中一维数组名会转换为指向数组第一个元素的指针，二维数组也是一样的，所以也会被转换为指向第一个元素的指针，它的第一个元素是一个数组，所以最终的结果就是二维数组名被转换成指向数组的指针

关于二维数组的数组名：

int (*p)[100] = m 中，对于 m[n] ，是 m 数组的第 n+1个元素的名字，也就是数组名

所以可以有 p=&m[n] ，其中p指向了一个数组元素，也就是矩阵中的某一行

当对 m[n] 进行 sizeof 操作时，结果是 100*sizeof(int) ；而 sizeof(m) 结果是50*100*sizeof(int)

指针数组、数组指针

明确运算符优先顺序：() > [] > *

指针数组

例子：int *p[100]

根据优先级，先看[]，则 p 是一个数组，该数组是一个 int * 型，因此数组的元素是指针类型，共 100 个元素。称为“指针的数组”，即指针数组

数组指针

行指针是一个指向一维数组的指针，它专门用来操作二维数组中的”整行”数据

例子：int (*p)[100]

根据优先级，先看括号内，可知 p 是一个指针，这个指针指向一个一维数组，数组长度为 100 。称为“指向数组的指针”，即数组指针

字符串与指针（了解）

char *cp = "abcdefg";中”abcdefg”会被转换为一个没有名字的字符数组，这个数组被 abcdefg 和一个空字符 ‘/0’ 初始化，并且会得到一个指针，它的值为第一个字符的地址

可以快速初始化字符数组：char ca[]="abcdefg";

两个方法的对比：

用来初始化字符数组的字符串常量，编译器会在栈中为字符数组分配空间，然后把字符串中的所有字符复制到数组中

用来初始化字符指针的字符串常量会被编译器安排到只读数据存储区，但也是按字符数组的形式来存储的，可以通过一个字符指针读取字符串常量但不能修改它，否则会发生运行时错误

不是所有的字符串常量都放在不同的地址

str1 和 str2 在栈中各自有不同的空间，而 str3 和 str4 是两个字符指针，分别被初始化为相同字符的字符串常量，二者指向相同的区域

指针的值传递（了解）

  1. a是一个字符指针，拿到的是字符串常量的第一个字符的地址，（这个地址是常量，但a不是）
  2. 将 a 的存储地址给到函数pointer_plus，将 a 所指向的地址向后推了3位=在主函数里a+=3;
  3. 所以可通过传递指针的指针（复制的），来改变原指针所指向的位置（被复制的改了）

动态内存分配

定义变量、数组，都是为存储对象静态分配内存

动态分配和静态分配的区别：

  1. 分配时机：

    静态分配在编译期就确定内存的大小和位置；动态则是在程序运行时根据需要来分配或释放内存

  2. 管理方式：

    静态分配由编译器自动管理，程序员无法干预；动态则需要手动管理

malloc 和 free

C标准库里用这两个函数来动态分配内存和释放、共同维护一个内存池

一、malloc函数

向内存申请一块连续可用的空间，并返回指向块开头的指针

void *malloc(size_t size)

函数动态分配size字节的内存块，类型size_t是用来表示内存大小的无符号整型，若内存分配失败，则返回NULL；若分配成功，则返回内存块的起始地址（基地址）

注意：

  1. malloc的返回值是void*型，强转是不必要的，推荐加上
  2. 分配的内存大小可以用 sizeof（） 乘所需要的类型的个数

二、free函数

动态分配的内存不会自动释放，必须由程序员调用free函数释放

void free(void *ptr)

  1. free 函数只能释放动态分配的内存块
  2. 如果 ptr 是零指针（空指针），则 free 函数不运行操作
  3. 函数将释放 ptr 指向的内存块

calloc和realloc

一、calloc函数

void* calloc (size_t num,size_t size);

num 是要分配的元素个数，size 是每个元素的字节数，它会在内存中分配 num * size 字节的连续空间

二、realloc函数

void* realloc(void* ptr, size_t size);

重新分配已经存在的内存的空间大小

ptr 表示要重新分配大小的指针，size 表示重新分配内存的大小，

  • 分配成功返回一个指针，不成功返回NULL
  • 重新分配的大于原来大小，原来的值会保留
  • 如果小于原来大小，多于的会被释放掉
